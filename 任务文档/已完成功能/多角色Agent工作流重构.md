# 多角色Agent工作流重构

## 目的
解耦并规范化生成应用程序时的代码结构。使得代码在输出前经过合理的架构设计、前后端开发职责分离以及 QA 把控。

## 解决机制
我们在 `src/app/api/chat/route.ts` 通过修改 `systemPrompt` 构建了一套 **“主代筹划，子代干活”** 的强制性串行执行流：
1. `product_manager` 会先根据自然语言拆解出结构化的 PRD (产品需求文档)。
2. `frontend_developer` 只关注视图层逻辑并应用现有脚手架规范。
3. `backend_developer` 只关注持久层和接口的数据交换逻辑。
4. `test_engineer` 作为质量阀门审查前三步的产出物。

修改了系统 Prompt 之后执行了标准的预发布验证，已在远端环境 `http://34.72.125.220:3000` 构建生效。
