# 子代理中文名称显示

## 目标
将子代理运行时的状态面板中的名称由默认的 "Sub-Agent" 修改为符合角色的中文名称（如“产品经理”、“前端开发”等）。

## 实现步骤
1. 修改了后台调度系统 prompt (`src/app/api/chat/route.ts`)：
   - 更新 `MANDATORY DEVELOPMENT WORKFLOW` 约束，在传递 agent 角色信息时强制包含明确的 `name` 属性（如 `name: "产品经理"`）。
   - 为子代理系统工具 `dispatch_subagent` 的 `Zod` 参数校验补充了 `name` 字段支持。
   
2. 修正了前端状态面板的解析逻辑 (`src/lib/log-parser.ts`)：
   - 编写了 `getAgentName` 解析助手。
   - 对历史缺省的日志数据做前向兼容，自动根据 `role` 映射中文（例如将 `product_manager` 或 `Product Manager` 智能转换成 "产品经理"）。
   
3. 更新了 UI 组件 (`src/components/chat/ai-status-panel.tsx`)：
   - 修改 `AgentCard` 的数据抽取方式，以正确显示 Agent 中文名称。
   - 修正了排版展示细节。

## 验证
执行 `npm run build` 并通过了本地构建，同时对包含模板字符串注入错误的语法进行了清理恢复。
